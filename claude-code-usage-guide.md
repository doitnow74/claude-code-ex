# 클로드 코드(Claude Code) 사용법 가이드

## 1. 개요

Claude Code는 Anthropic이 제공하는 CLI 기반 AI 코딩 어시스턴트입니다. 터미널, 데스크톱 앱(Mac/Windows), 웹 앱(claude.ai/code), IDE 확장(VS Code, JetBrains) 등 다양한 환경에서 사용할 수 있습니다.

## 2. 기본 사용법

- 터미널에서 `claude` 명령으로 실행
- 자연어로 작업을 요청하면 코드 탐색, 수정, 실행, 디버깅 등을 자동으로 수행
- 파일 읽기/쓰기, 셸 명령 실행, 검색(Glob/Grep), 웹 검색 등 다양한 도구를 활용해 작업 수행

## 3. 주요 슬래시 명령어

| 명령어 | 설명 |
|---|---|
| `/help` | 도움말 보기 |
| `/init` | 현재 코드베이스를 문서화한 CLAUDE.md 생성 |
| `/review` | PR(Pull Request) 리뷰 |
| `/code-review` | 현재 변경사항(diff)에 대한 코드 리뷰 (effort 단계별: low/medium/high/ultra) |
| `/security-review` | 보류 중인 변경사항에 대한 보안 검토 |
| `/run` | 앱을 실행하고 동작 확인 |
| `/schedule` | 반복 실행되는 예약 작업(클라우드 에이전트) 생성 |
| `/loop` | 특정 명령을 주기적으로 반복 실행 |
| `/agents` | 서브에이전트 설정 (터미널 대화형 패널) |
| `/config` | 설정 변경 (모델, 테마 등) |
| `/permissions` | 권한 설정 관리 |
| `/hooks` | 훅(Hook) 설정 관리 |
| `/doctor` | 환경 진단 |

## 4. CLAUDE.md 파일

- 프로젝트 루트에 두는 컨텍스트 파일로, 코드베이스 구조, 컨벤션, 빌드/테스트 명령 등을 기록
- Claude Code가 매 세션마다 자동으로 참고함
- `/init`으로 최초 생성, 이후 직접 수정하거나 관련 스킬로 개선 가능

## 5. 권한 모드와 안전장치

- 파일 수정, 명령 실행 등 위험도가 있는 작업은 사용자 승인을 거침
- 권한 모드를 통해 자동 승인 범위를 설정할 수 있음 (`/permissions` 또는 `settings.json`)
- 파괴적 작업(강제 푸시, `rm -rf`, DB 테이블 삭제 등)은 기본적으로 사전 확인을 거치도록 설계됨

## 6. 서브에이전트(Agent)

복잡하거나 범위가 넓은 작업은 전문화된 서브에이전트에게 위임할 수 있습니다.

- **Explore**: 코드 위치 탐색 전용 (읽기 전용)
- **general-purpose**: 복잡한 다단계 작업, 광범위한 검색
- **Plan**: 구현 전략 설계
- **claude-code-guide**: Claude Code/Agent SDK/API 사용법 질문

## 7. 메모리(Memory) 시스템

- 세션 간 지속되는 파일 기반 메모리로 사용자 선호, 피드백, 프로젝트 컨텍스트 등을 저장
- 사용자(user), 피드백(feedback), 프로젝트(project), 참조(reference) 4가지 유형으로 구분
- `MEMORY.md` 인덱스를 통해 관리

## 8. 스킬(Skill)

- `/스킬이름` 형태로 호출 가능한 사전 정의된 작업 모듈
- 예: `docx`, `pptx`, `xlsx`, `pdf` 파일 작업, `code-review`, `security-review`, `schedule`, `loop` 등
- 사용자가 명시적으로 호출하거나 요청 내용과 일치할 때 자동 실행

## 9. 모범 사용 팁

1. **명확한 요청**: 모호한 지시보다 구체적인 파일/함수/목표를 명시하면 정확도가 높아짐
2. **작은 단위로 검증**: 변경 후 테스트/타입체크를 실행해 회귀를 방지
3. **계획 모드 활용**: 복잡한 작업 전 Plan으로 설계를 먼저 합의
4. **CLAUDE.md 유지보수**: 프로젝트 컨벤션이 바뀌면 함께 업데이트
5. **위험한 작업은 재확인**: 강제 푸시, 브랜치 삭제 등은 사용자 승인 필수임을 인지
